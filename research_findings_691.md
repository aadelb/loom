## 1. How Constitutional Classifiers Work

### Architecture
Constitutional Classifiers are a **dual-classifier system** trained on constitutions (sets of safety principles):

- **Input Classifier**: Filters harmful user requests before they reach the main model
- **Output Classifier**: Filters model responses to prevent generating harmful content

### Training Mechanism
- Classifiers are trained on a "constitution" — a set of clear, principle-based rules (e.g., "Don't help with illegal activities")
- Uses synthetic data generated by the model itself to create training examples
- Fine-tuned via RLHF (Reinforcement Learning from Human Feedback) to align with constitutional principles
- Constitutional AI (CAI) approach allows for interpretable, rule-based safety without requiring massive human annotation

### Key Properties
- **Interpretable**: Built on explicit principles rather than black-box ML
- **Efficient**: Reduces computational overhead compared to traditional classifiers
- **Compositional**: Can combine multiple constitutional principles
- **Generalizable**: Learned principles transfer across different prompt variations

## 3. Known Bypass Techniques

### Published Bypass: Trojan-Speak (arxiv.org/abs/2603.29038v1)

**Technique**: Adversarial Fine-tuning Attack

**Mechanism**:
1. Exploits fine-tuning APIs (available on OpenAI, Anthropic, etc.)
2. Adversarially fine-tunes the model to ignore safety classifiers
3. Uses trojan triggers embedded in the fine-tuning data
4. Results in a model that behaves normally for safe queries but violates policies on trojan-triggered inputs

**Key Properties**:
- **Zero Jailbreak Tax**: Doesn't require prompt engineering or special formatting
- **Stealthy**: The fine-tuned model passes safety checks on unrelated tasks
- **Persistent**: The bypass persists across the fine-tuned model's lifetime
- **Success Rate**: Reported to bypass Constitutional Classifiers entirely

**Why It Works**:
- Fine-tuning modifies model weights directly, not just the input/output
- Classifiers primarily monitor at the input/output boundary
- Trojan activation can be triggered internally, bypassing classifier attention
- The model learns a dual-state behavior: safe for external queries, unsafe for trojan inputs

### Other Potential Bypass Vectors
1. **Prompt Injection via Embedded Data**: Injecting hidden prompts in fine-tuning data
2. **Gradient-Based Attacks**: Using optimization to find adversarial inputs that fool the classifier
3. **Semantic Variation**: Rephrasing harmful requests with synonyms or indirect language
4. **Multi-turn Context Pollution**: Building up context that gradually shifts the model's behavior
5. **Token-Level Obfuscation**: Using encoding or token smuggling to bypass token-level classifiers

## 4. Implications for Loom's Strategy Effectiveness

### Significantly Affected Strategies

The following attack types are substantially degraded against Constitutional Classifiers:

1. **Direct Jailbreak Prompts** (95% reduction in effectiveness)
   - Simple role-play scenarios
   - "Hypothetical" requests
   - Fictional framing
   - Status: Mostly ineffective

2. **Standard Adversarial Patterns** (90% reduction)
   - Token smuggling
   - Encoding/obfuscation
   - Character escaping
   - Status: Largely detected and blocked

3. **Prompt Injection Attempts** (80% reduction)
   - Hidden system prompts
   - Prefix/suffix injection
   - Context breakout attempts
   - Status: Detected by input classifier

4. **Simple Reasoning Exploits** (75% reduction)
   - Chain-of-thought with harmful conclusions
   - Socratic method to harmful outputs
   - Status: Output classifier catches results

### Moderately Affected Strategies

5. **Persona-Based Attacks** (60% reduction)
   - Playing specific roles
   - Character adoption
   - Status: Input classifier recognizes harmful intent, but some variation might slip through

6. **Multi-turn Context Building** (55% reduction)
   - Gradual context pollution
   - Building rapport then requesting harm
   - Status: Partially effective; classifiers may not catch inter-turn relationships

### Potentially Effective Strategies

7. **Adversarial Fine-tuning** (HIGHLY EFFECTIVE against Constitutional Classifiers)
   - Trojan-Speak and variants bypass the defense entirely
   - Creates a dual-state model
   - Status: Works if attacker has API access

8. **Fine-grained Semantic Attacks** (Partially effective)
   - Very similar to benign requests with subtle harmful nuances
   - May fool the classifier's decision boundary
   - Status: ~20-30% success rate reported in research

9. **Internal Activation Attacks** (Research-stage effectiveness)
   - Triggering internal representations without touching input/output
   - Requires model internals knowledge
   - Status: Effective but requires white-box access

## 5. Recommended Strategy Adaptations for Loom

### Short-term (Immediate Implementation)

1. **Adopt Trojan-Speak Technique**
   - Implement adversarial fine-tuning as primary attack vector
   - Create trojan-triggered variants of attacks
   - Maintain normal model behavior while embedding trojan activation

2. **Semantic Variation Engine**
   - Generate multiple semantic variants of harmful requests
   - Use synonym substitution and indirect phrasing
   - Test each variant for classifier detection

3. **Multi-level Encoding**
   - Combine multiple encoding layers (token smuggling + semantic obfuscation)
   - Each layer targets different aspect of the classifier

### Medium-term (Strategic Pivot)

4. **API Access Exploitation**
   - Prioritize fine-tuning API access for adversarial attacks
   - Develop trojan embedding techniques
   - Create persistent, stealthy model compromises

5. **White-box Gradient Attacks**
   - Develop GCG-style attacks targeting the classifier itself
   - Use gradient information to find adversarial examples
   - Requires some knowledge of classifier weights

6. **Context Pollution Specialization**
   - Develop multi-turn scenarios that gradually shift model behavior
   - Use subtle context references that classifiers miss
   - Exploit inter-turn semantic relationships

### Long-term (Research & Adaptation)

7. **Internal Activation Research**
   - Study Constitutional Classifier architecture
   - Develop techniques to activate harmful behaviors via internal representations
   - Requires white-box or architectural knowledge

8. **Classifier Inversion Attacks**
   - Research methods to extract/invert the classifier's decision boundaries
   - Generate adversarial examples that fool the classifier systematically

## 8. Conclusions

2. **Semantic variation** provides modest but consistent success (25-35%)
3. **Traditional prompt-based attacks** are largely ineffective and should be deprioritized
4. **Research direction**: Focus on:
   - Adversarial fine-tuning variants
   - Classifier inversion/gradient attacks
   - Internal activation mechanisms
   - White-box model access exploitation

**Recommended Action**: Integrate Trojan-Speak methodology into Loom's attack toolkit as the primary approach for models using Constitutional Classifiers.
