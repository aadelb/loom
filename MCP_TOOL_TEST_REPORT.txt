================================================================================
ROOT CAUSE ANALYSIS
================================================================================

The 93 TaskGroup errors appear to stem from one of these patterns:

1. Async Context Issues
   - Tools trying to instantiate async resources in sync context
   - Missing proper async/await handling
   
2. Dependency Initialization Failures
   - External services unreachable (Tor, transcription APIs, etc.)
   - Missing API keys or configuration
   - Network timeouts during initialization
   
3. Exception Handling Gaps
   - Exceptions not properly caught in TaskGroup subtasks
   - Unhandled exceptions bubbling up as TaskGroup failures
   
Recommended Fix: Add try/except blocks around all TaskGroup creation
and add proper logging to identify which subtask is failing.

TEST METHODOLOGY
================================================================================

Script: test_all_mcp_tools.py
Method:
1. Connect to MCP server (127.0.0.1:8787/mcp)
2. List all registered tools via ListTools RPC
3. For each tool:
   a. Generate minimal valid arguments from schema
   b. Create fresh MCP session
   c. Invoke tool with 30-second timeout
   d. Record result (status, size, error, time)
   e. Close session
4. Aggregate results and produce report

Argument Generation Strategy:
- string: "test" or "https://example.com" (for URLs)
- integer: 5
- number: 3.14
- boolean: True
- array: ["test"] or [1]
- object: {}

Session Management:
- One session for listing tools (reused)
- Fresh session for each tool (prevents state issues)
- Automatic cleanup on success or error
