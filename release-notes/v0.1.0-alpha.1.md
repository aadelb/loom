# Loom v0.1.0-alpha.1 Release

## Overview

Loom v0.1.0-alpha.1 is the initial research release of the Loom MCP server — a multi-LLM orchestration platform for tool-augmented AI agents. This alpha focuses on provider stability, safety, and concurrent load resilience across NVIDIA NIM, OpenAI, Anthropic, and local vLLM backends.

## Major Fixes (Wave 3 Cross-Model Review)

### CRITICAL

1. **LLM Embed Double-Invocation**
   - Fixed `research_llm_embed` calling `await p.embed()` twice (inside cascade loop and again on selection), halving embedding latency and cost
   - Provider override path now invokes embed exactly once

2. **Whitespace-Only API Keys Accepted**
   - `nvidia_nim.py`, `openai_provider.py`, `anthropic_provider.py` now validate `bool(api_key and api_key.strip())`
   - Prevents false "available" advertising and 401 traceback leaks

3. **CostTracker Race Condition**
   - Eliminated read-modify-write without locks (concurrent calls could lose cost data)
   - Added `threading.Lock` (in-process) + `fcntl.flock` (cross-process)
   - Verified: 5 threads × 20 add_cost calls → 100/100 persisted
   - Cap check moved before persistence to prevent polluted disk state

4. **Journey Callback Error Handling**
   - All 9 `on_step(step_N)` call sites now exception-wrapped
   - User callback errors (closed file, typo) no longer kill the entire journey
   - Added `_safe_on_step()` helper with logging

### HIGH

5. **Provider Exception Logs Sanitized**
   - OpenAI & Anthropic exceptions now filter API keys before logging and re-raising
   - Regex replaces `sk-`/`sk-ant-`/`nvapi-` prefixes with `[REDACTED_KEY]`

6. **vLLM/NIM Embed Timeout Hardcoding**
   - Both providers now accept `timeout` kwarg (no longer hardcoded 60s)
   - Threaded to httpx layer

7. **NIM Embed Rate-Limit Bypass**
   - `NvidiaNimProvider.embed()` now respects 12-parallel semaphore cap
   - Prevents rate-limit spikes from embedding requests

8. **Per-Call Timeout Not Threaded**
   - OpenAI: `timeout` now passed to `client.chat.completions.create()`
   - Anthropic: `timeout` now passed to `client.messages.create()`
   - Embeddings path also threads timeout parameter

- Async I/O only (no synchronous API)

  - NVIDIA NIM (Kimi K2.5, Llama 4, Mistral Large 3, Qwen 3.5)

  - Anthropic (Claude 3 Sonnet)

## Security

- Secrets never logged; exceptions sanitized
- Concurrent cost accounting protected by dual-lock strategy
- SSRF tests pass (no unintended HTTP escalation)
