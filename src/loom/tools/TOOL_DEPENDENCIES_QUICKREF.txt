================================================================================
WHAT IT DOES
================================================================================

Wires research tool dependencies into the pipeline composer for automatic:
1. Dependency resolution (find all prerequisites)
2. Execution planning (organize into parallel groups)
3. Topological sorting (ensure correct execution order)
4. Cycle detection (validate dependency graph is acyclic)
5. Cost tracking (estimate & measure execution cost)
6. Parallel execution (run independent tools simultaneously)

================================================================================
INTEGRATION FUNCTIONS (Used by orchestrator/enhancer)
================================================================================

1. prepare_tool_execution(tools: list[str]) -> dict
   ├─ Main hook for pipeline composer
   ├─ Resolves dependencies and returns execution plan
   ├─ Returns: execution_plan, first_group, remaining_groups, valid
   └─ Called by: pipeline_enhancer.py, orchestrator.py

2. get_execution_plan(tools: list[str]) -> list[list[str]]
   ├─ Compute parallel groups in topological order
   ├─ Returns: [group0, group1, ...] where group = [tool1, tool2, ...]
   └─ Group 0 tools execute in parallel, then group 1, etc.

3. resolve_dependencies(tools: list[str]) -> set[str]
   ├─ Transitively resolve all prerequisites (BFS)
   ├─ Returns: set of all tool names (input + prerequisites)
   └─ Example: {"research_deep", "research_search", "research_fetch", ...}

4. validate_execution_order(plan: list[list[str]]) -> dict
   ├─ Validate execution plan respects all dependencies
   ├─ Returns: {valid: bool, violations: list, issues: list}
   └─ Call before executing a plan if needed

================================================================================
EXECUTION PLANNING ALGORITHM
================================================================================

Input: List of tools (e.g., ["research_deep"])

Step 1: Resolve Dependencies
  └─ BFS to find all prerequisites
  └─ Result: {"research_deep", "research_search", "research_fetch", "research_markdown"}

Step 2: Compute Depths
  └─ Distance from leaf tools
  └─ research_search/fetch/markdown = depth 0 (leaf)
  └─ research_deep = depth 1 (depends on depth 0)

Step 3: Group by Level
  └─ Group 0: [research_search, research_fetch, research_markdown]
  └─ Group 1: [research_deep]

Step 4: Return Groups in Order
  └─ Execution: Run Group 0 in parallel, then Group 1

Result: Topologically sorted execution plan with parallelization
