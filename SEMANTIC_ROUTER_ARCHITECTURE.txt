┌─────────────────────────────────────────────────────────────────────────────┐
│ EMBEDDING CACHE LIFECYCLE                                                   │
└─────────────────────────────────────────────────────────────────────────────┘

1. TOOL DISCOVERY (First call only)
   ├─ Scan /Users/aadel/projects/loom/src/loom/tools/*.py (360+ files)
   ├─ Parse each file with ast.parse()
   ├─ Extract research_* function definitions
   ├─ Extract first line of docstrings as descriptions
   └─ Build dict: {"research_search": "Search...", ...}

2. CACHE LOAD CHECK (Startup)
   ├─ Check ~/.cache/loom/tool_embeddings.npy exists?
   │  ├─ Yes: Load embeddings array
   │  └─ Verify shape matches tool count
   ├─ Check ~/.cache/loom/tool_names.npy exists?
   │  └─ Verify tool names match current set
   └─ If cache valid: Use cached embeddings
      If cache invalid: Rebuild (proceed to step 3)

3. EMBEDDING GENERATION (First call or cache miss)
   ├─ Try sentence_transformers.SentenceTransformer("all-MiniLM-L6-v2")
   │  ├─ Success: Encode all tool descriptions
   │  │  └─ Output: (num_tools, 384) float32 array
   │  └─ Failure: Try sklearn fallback
   │
   ├─ Try sklearn.feature_extraction.text.TfidfVectorizer
   │  ├─ Success: Fit on tool descriptions
   │  │  └─ Output: (num_tools, max_features) sparse array
   │  └─ Failure: Use keyword fallback
   │
   └─ If all fail: Warn but continue (keyword only)

4. CACHE PERSISTENCE (After generation)
   ├─ Create ~/.cache/loom/ directory (if needed)
   ├─ Save embeddings as tool_embeddings.npy
   │  └─ Binary numpy format (1.3 MB for 350 tools)
   ├─ Save tool names as tool_names.npy
   │  └─ Object array of strings
   └─ Set appropriate file permissions

5. USAGE (Query time)
   ├─ Embed query using same method
   ├─ Compute cosine_similarity(query_emb, all_embeddings)
   ├─ Get top-10 tools by similarity
   └─ Return with similarity scores


╔════════════════════════════════════════════════════════════════════════════╗
║                       FALLBACK STRATEGY                                    ║
╚════════════════════════════════════════════════════════════════════════════╝

TIER 1: Sentence-Transformers (Primary)
├─ Model: all-MiniLM-L6-v2 (45 MB)
├─ Embedding Dims: 384
├─ Speed: ~100 tokens/sec
├─ Accuracy: Highest
├─ First Run: ~2-5s (download + load)
├─ Subsequent: ~50-100ms per query
└─ Requirement: sentence-transformers library

TIER 2: Sklearn TF-IDF (Fallback 1)
├─ Method: TfidfVectorizer with bigrams
├─ Embedding Dims: ~500 sparse
├─ Speed: <50ms
├─ Accuracy: Good
├─ First Run: Build vectorizer from descriptions
├─ Subsequent: <50ms per query
└─ Requirement: scikit-learn library

TIER 3: Keyword Matching (Fallback 2)
├─ Method: Simple token overlap
├─ Processing: Text tokenization + set intersection
├─ Speed: <10ms
├─ Accuracy: Basic
├─ Requirements: None (Python stdlib only)
└─ Output: Normalized keyword overlap ratio
