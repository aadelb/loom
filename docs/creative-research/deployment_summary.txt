ARCHITECTURE DETAILS
====================

Key Components:

  QueryTest
    Dataclass holding query metadata
    Fields: query, expected_types, description

  run_deep_research()
    Async wrapper around research_deep()
    Fixed parameters: depth=1, no LLM features
    Type-safe interface to deep pipeline

  create_mock_response()
    Generates realistic mock response structures
    Auto-calculates provider lists by query type
    Simulates expected pages and costs

  mock_deep_research()
    Replaces research_deep in --mock mode
    Uses actual _detect_query_type() function
    Simulates 500ms network latency

  print_result_summary()
    Formats query result with visual indicators

    Reports all metrics and warnings

  run_stress_test()
    Main orchestration loop
    Runs all 5 tests sequentially
    Handles both mock and live modes
    Accumulates statistics

    Entry point with argparse CLI
    Handles --mock, --verbose flags
    Returns proper exit codes
