DESCRIPTION:
  Aggregates vulnerability intelligence from 6 free sources to provide
  comprehensive CVE and security advisory data with deduplication, severity
  ranking, and exploit availability tracking.

DATA SOURCES (6 APIs):
  1. NVD (National Vulnerability Database)
     - URL: https://services.nvd.nist.gov/rest/json/cves/2.0
     - Provides CVSS scores, severity, descriptions
  
  2. GitHub Security Advisories
     - URL: https://api.github.com/advisories
     - Covers GitHub-tracked vulnerabilities and advisories
  
  3. CISA Known Exploited Vulnerabilities
     - URL: https://www.cisa.gov/sites/default/files/feeds/...
     - Real-world exploitation tracking
  
  4. Vulners API
     - URL: https://vulners.com/api/v3/search/lucene/
     - Aggregated vulnerability database
  
  5. GitHub PoC Search
     - URL: https://api.github.com/search/repositories
     - Public proof-of-concept repositories
  
  6. (Exploit-DB) - Ready for integration
     - Note: Requires HTML parsing due to API limitations

FUNCTION SIGNATURE:
  def research_vuln_intel(query: str, max_results: int = 30) -> dict[str, Any]

PARAMETERS:
  - query (str): Vulnerability search keyword/phrase
                 Examples: "Log4j", "OpenSSL", "SQL injection"
  - max_results (int): Maximum results (default 30, clamped 1-100)

RETURN VALUE:
  {
    "query": str,                    # Original search query
    "sources_checked": list[str],    # Sources queried (5 sources)
    "total_vulns": int,              # Total unique vulnerabilities found
    "vulns": [
      {
        "source": str,               # Source database name
        "id": str,                   # Vulnerability ID (CVE or advisory ID)
        "title": str,                # Short description (<=100 chars)
        "severity": str,             # CRITICAL/HIGH/MEDIUM/LOW/UNKNOWN
        "description": str,          # Full description (<=300 chars)
        "url": str,                  # Reference URL
        "published": str,            # Publication date (ISO 8601)
        "exploits_available": bool,  # Has public exploit/PoC
        "cvss_score": float|None,    # CVSS score if available
      },
      ...
    ],
    "error": str  # Optional: error message if query invalid
  }
