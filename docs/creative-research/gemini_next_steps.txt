### 1. Fix Zero-Cost Products & Set Costing Method (Margin Protection)
**Why it's critical:** If your system thinks an item costs 0 AED, your Profit & Loss reporting is useless, and cashiers or managers might accidentally approve discounts that result in negative margins. You've already done the hard work of extracting 18 POs via OCR; you need to map that data back to the products.
**Action:** 
- Ensure the `stock_account` module is installed.
- Set the Product Category Costing Method (`property_cost_method`) to **Average Cost (AVCO)**.
- Run a script via the Odoo XML-RPC/External API to backfill the `standard_price` on products using the unit prices from your existing `purchase.order.line` records.

**API Example (Python xmlrpc.client):**
```python
# Find PO lines for products with 0 cost to backfill standard_price
zero_cost_products = models.execute_kw(db, uid, pass, 'product.product', 'search_read', 
    [[('standard_price', '=', 0)]], 
    {'fields': ['id', 'name']}
)

# For each product, find the latest purchase order line and update the cost
for p in zero_cost_products:
    po_line = models.execute_kw(db, uid, pass, 'purchase.order.line', 'search_read',
        [[('product_id', '=', p['id'])]],
        {'fields': ['price_unit'], 'order': 'id desc', 'limit': 1}
    )
    if po_line:
        models.execute_kw(db, uid, pass, 'product.product', 'write', 
            [[p['id']], {'standard_price': po_line[0]['price_unit']}]
        )
```

### 2. Configure POS with Cashier Restrictions (Theft/Fraud Prevention)
**Why it's critical:** In small supermarkets, the highest risk of unseen loss is at the register (sweet-hearting, unauthorized voids, price overrides). Your Dahua CCTV is useless if you don't know *when* to check the footage.
**Action:** 
- Install `point_of_sale`.
- Go to POS Settings and strictly lock down permissions. Enable **"Restrict Price Modification"** and require a **"Manager PIN"** for line voids, order cancellations, and applying discounts. 
- When an anomaly occurs in your daily POS Z-report (e.g., an unusual number of voids), you can look at the exact Odoo POS timestamp and match it with the Dahua XVR footage.

### 3. Setup Scrap Locations & Expiry Alerts (Shrinkage Management)
**Why it's critical:** You have `product_expiry` enabled, which is great for dairy and fresh goods. However, if an item expires or a glass jar breaks, and you just "throw it away" without telling Odoo, your inventory stays artificially high. Odoo will think you have 5 bottles of milk, won't trigger a reorder, and a customer will face an empty shelf.
**Action:** 
- Go to Inventory -> Configuration -> Locations. Create a Virtual Location named "Damaged/Expired (Scrap)" and check the **"Is a Scrap Location"** box.
- Configure Odoo to generate an Activity (Notification) for the store manager based on the `alert_date` on the `stock.lot` (e.g., 3 days before milk expires) so you can discount it and move it to a clearance bin before it becomes a total loss.

### 4. Assign Internal Barcodes to the 32 Missing Products (Operational Bottleneck)
**Why it's critical:** If a product doesn't have a barcode, the cashier has to type the name into the POS. This slows down the line, frustrates customers, and leads to inventory errors (e.g., selecting "Lipton Tea 250g" instead of "Lipton Tea 500g").
**Action:**
- Filter for products where `barcode` is False. 
- If they are items without manufacturer barcodes (e.g., fresh produce, loose bakery items), Odoo can generate internal EAN13 barcodes.
- Print these using Odoo's built-in label layout (`action_open_label_layout`) and stick them on the shelves or directly on the items.

**API Example:**
```python
# Find missing barcodes
missing = models.execute_kw(db, uid, pass, 'product.product', 'search', [[('barcode', '=', False)]])
# You can generate a sequence and write it back to the 'barcode' field.
```

### 5. Setup Reordering Rules (Min/Max) for the Top 50 "Essentials" Only
**Why it's critical:** You don't need reordering rules for all 992 products right now—managing that out of the gate is overwhelming. However, running out of "destination items" (Al Ain Water, Almarai Milk, fresh bread, eggs) will cause customers to leave and go to the baqala next door.
**Action:**
- Identify your 50 most essential, fastest-moving items.
- Create `stock.warehouse.orderpoint` records for them. Set the `product_min_qty` to a 2-day supply and `product_max_qty` to a 5-day supply. 
- When stock drops below the minimum, Odoo will automatically draft a Purchase Order, which you can easily send to the supplier via WhatsApp.

**API Example:**
```python
# Create an orderpoint for a fast-moving item
models.execute_kw(db, uid, pass, 'stock.warehouse.orderpoint', 'create', [{
    'product_id': 123, # e.g., Al Ain Water 1.5L
    'product_min_qty': 24.0, # 4 packs of 6
    'product_max_qty': 60.0, # 10 packs of 6
    'location_id': 8 # Your main stock location ID
}])
```
