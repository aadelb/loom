CORE FUNCTIONALITY:
- Builds relationship graphs from public data across 4 platforms:
  * GitHub: Co-contributor relationships (analyzes repos and contributors)
  * Reddit: Community engagement and user mentions 
  * HackerNews: Topic interests based on submission history
  * Semantic Scholar: Co-authorship networks

FUNCTION SIGNATURE:
  research_social_graph(username: str, platforms: list[str] | None = None) -> dict[str, Any]

PARAMETERS:
- username: Username/identifier to analyze (interpreted per platform)
- platforms: Defaults to ["github", "reddit", "hackernews"]
            Supports: "github", "reddit", "hackernews", "semanticscholar"

RETURN VALUE:
{
  "username": str,
  "nodes": [{"id": str, "platform": str, "name": str}, ...],
  "edges": [{"source": str, "target": str, "relationship": str, "weight": int}, ...],
  "platforms_analyzed": list[str],
  "total_connections": int
}

PLATFORMS ANALYZED:
1. GitHub: Uses public GitHub API v3
   - Fetches user repos (limit: 50)
   - Fetches repo contributors (limit: 30 per repo)
   - Builds edges: user->contributor, co-contributor pairs

2. Reddit: Uses public Reddit JSON API
   - Fetches recent comments (limit: 100)
   - Extracts subreddit activity counts
   - Extracts u/ mentions from comment text
   - Builds edges: user->subreddit (activity count), user->mentioned user

3. HackerNews: Uses Firebase API
   - Fetches user profile and karma
   - Fetches submissions (limit: 50)
   - Extracts topics from titles and URLs
   - Builds edges: user->topic (interest count)

4. Semantic Scholar: Uses public API
   - Searches for author by name
   - Fetches author papers (limit: 30)
   - Extracts co-authors from papers
   - Builds edges: author->co-author (paper count)

LIMITATIONS:
- HN and Reddit rate limits: No built-in delays (public APIs)
- GitHub: 1 req/sec enforced (public API limit)
- Semantic Scholar: Internal rate limiting
- Missing live API testing (uses mocked tests)
- Large networks may be truncated to prevent timeout
