=== INFRASTRUCTURE ANALYSIS TOOLS IMPLEMENTATION ===

Three new tools have been successfully implemented and integrated into the Loom MCP server:

1. RESEARCH_REGISTRY_GRAVEYARD
   Location: /Users/aadel/projects/loom/src/loom/tools/infra_analysis.py
   
   Scans package registries for deleted/yanked packages and typosquatting risks:
   - PyPI: Checks for yanked versions
   - NPM: Checks for deprecated versions
   - RubyGems: Checks for yanked packages
   
   Features:
   - Shannon entropy calculation for package names (detects random/suspicious names)
   - Typosquatting candidate detection using Levenshtein distance
   - Risk level assessment (critical/high/medium/low)
   - Returns: package exists status, yanked count, entropy score, similar names
   
   Parameters: package_name (str), ecosystem (Literal["pypi", "npm", "rubygems"])

2. RESEARCH_SUBDOMAIN_TEMPORAL
   Location: /Users/aadel/projects/loom/src/loom/tools/infra_analysis.py
   
   Tracks subdomain births/deaths over time via Certificate Transparency logs:
   - Uses crt.sh to retrieve all certificates
   - Groups certificates by date and detects patterns
   - Flags internal tool exposure (jira, jenkins, grafana, etc.)
   - Detects certificate issuance bursts (potential infrastructure changes)
   
   Features:
   - Monthly distribution tracking
   - Internal tools detection (critical risk indicator)
   - Burst pattern detection (>2.5x average certs in a month)
   - Geographic diversity analysis via certificate issuers
   - Returns: subdomain count, burst detection, internal tools exposed, monthly stats
   
   Parameters: domain (str), days_back (int, 1-365, default 90)

3. RESEARCH_COMMIT_ANALYZER
   Location: /Users/aadel/projects/loom/src/loom/tools/infra_analysis.py
   
   Analyzes GitHub commit patterns for intelligence signals:
   - Detects security-focused commits (CVE, fix, patch, vuln patterns)
   - Calculates crunch score (% commits on weekends/nights)
   - Tracks author churn (new vs returning developers)
   - Analyzes sentiment from commit messages
   - Detects tech direction changes (dependencies, CI/CD, containerization)
   
   Features:
   - GitHub API integration (no auth required for public repos)
   - Multiple signal detection (security, infrastructure, dependencies)
   - Risk assessment based on security incidents and crunch patterns
   - Returns: security incidents, crunch score, sentiment trend, tech direction
   
   Parameters: repo (str, "owner/name"), days_back (int, 1-365, default 30)
