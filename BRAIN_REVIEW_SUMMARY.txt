   Location: All files calling _call_with_cascade()
   Problem: No provider_override="nvidia" → tries Groq, DeepSeek, then expensive providers
   Impact: 40% timeout rate, unexpected cost, slow performance

   Location: src/loom/brain/reasoning.py in select_tool()
   Problem: semantic_router returns non-existent tool names without validation
   Example: "research_analyze_code_quality" (doesn't exist)
   Impact: Execution crashes with "Tool not found"
   Fix: Validate tools against actual server registry before returning (~50 lines)

4. TIMEOUT MANAGEMENT
   Location: src/loom/brain/core.py in process() method
   Problem: 30s limit + 6-13s param extraction + 5-30s tool execution = 35-43s > 30s timeout
   Impact: Every complex request times out
   Fix: Allocate timeout budget per layer, fail fast on approaching limit (~100 lines)

5. NO EMBEDDINGS FALLBACK
   Location: src/loom/brain/memory.py in MemorySystem.__init__()
   Problem: Raises exception if sentence-transformers fails to load, no fallback
   Impact: Production deployment crashes on embeddings model load failure
   Fix: Add graceful fallback to keyword_match when embeddings unavailable (~80 lines)

6. ORCHESTRATION ERROR CASCADE
   Location: src/loom/brain/core.py in process() method
   Problem: Failures in one layer crash entire pipeline, no error recovery
   Impact: No graceful degradation, mysterious timeout errors
   Fix: Implement try/catch recovery at each layer with sensible fallbacks (~120 lines)
